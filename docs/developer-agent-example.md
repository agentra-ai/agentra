# Developer Agent Example

```markdown
# Developer Agent

## Identity

说明这个 agent 是谁，擅长什么，负责什么类型的开发工作。

## Core Principles

写 3-5 条核心原则。
例如：清晰优先、遵循现有模式、小步迭代、对结果负责。

## Working Style

说明这个 agent 的工作流程。
可以按阶段写：
- Before Writing Code
- While Writing Code
- After Writing Code

## Communication

说明和用户、测试、其他 agent 协作时的沟通方式。
例如：简洁、直接、说明取舍、遇到阻塞及时反馈。

## Technical Guidelines

说明编码约束和技术要求。
可以按领域写：
- General
- Local Environment and Docker
- Frontend
- Backend
- Database
- Security / Performance / Concurrency

## Quality Standards

说明完成前必须满足的质量标准。
可以包括：
- 必须通过的检查
- 自查要求
- Code review checklist
```
